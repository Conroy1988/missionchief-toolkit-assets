Quick Start

The shortest route from installation to a working command setup.

  1. Open the MissionChief map. The Toolkit adds its command button and dock directly to the map.
  2. Click the command button or press M to open the panel.
  3. Choose an Interface Theme, then choose an operational map skin. These settings are independent.
  4. Enable only the map overlays you need under Tools and Resources.
  5. Open Mission Age Watch with W when you need to triage ageing missions.
  6. Use the ? button in the panel header to reopen this guide.
A sensible first setup is to leave all visibility controls ON, enable only essential overlays, then save the complete arrangement as a Map Profile.

Mission Age Watch

The Toolkit’s main mission-triage dashboard.

Open with W or its command button. Mission Age Watch combines map missions, mission progress and personal vehicle data. While a required source is loading it uses neutral syncing or unknown states instead of claiming that zero units exist.

Sort, origin and cards

Sort by Age, Closest or Furthest. Distance can use the live map centre, a locked centre, a Quick Place or a saved bookmark. Lock Here freezes the current centre.

  • MISSION CLEARING is green and shows progress where available.
  • VEHICLES NEED ASSISTANCE is amber and can show missing personnel or vehicle requirements.
  • NO UNITS ON SCENE is shown only when source data is ready.
  • ON SCENE / STABLE remains separate from clearing.
  • ZOOM performs map tracking; OPEN opens the MissionChief mission.
  • Large result sets render progressively in batches.
Data sync is the latest source update. View updated is when the visible cards were rendered.

08

Resource and Transport Tools

Live mission requirements, coverage analysis, passive transport monitoring and manual alliance patient-release support.

Mission Requirements

The default-enabled matrix appears above MissionChief dispatch controls in normal document flow. It shows each supported requirement with Missing on mission, En-route, Still needed and Selected values, updating immediately when eligible vehicles are checked or unchecked.

  • Red means a known requirement is definitely still open.
  • Green means every displayed requirement is definitely covered by en-route and selected capacity.
  • Amber means personnel capacity, mission conditions or unknown MissionChief wording cannot be resolved exactly.
  • Normal and occupied vehicle lists are included. Trailer/towing pairs and capacity factors are deduplicated.
  • When the equivalent LSSM enhanced-missing-vehicles panel is active, the Toolkit yields rather than displaying a competing panel.

Resource Gap

Compares mission requirements with available personal vehicle context inside the selected radius. It can add a map badge and a requirement breakdown to Mission Inspector.

Transport Watcher

Shows amber patient or prisoner badges beside missions where MissionChief indicates transport is required.

Patient Transport Sweep

  1. Select Scan Transports to build the current alliance candidate queue.
  2. Review the delay and maximum-per-run settings, then start the sweep manually.
  3. The sweep opens one mission at a time, waits for delayed LSSM Release patient (No reward) controls and processes one eligible alliance-member ambulance at a time.
  4. The signed-in player's verified vehicles are excluded. Rows with ambiguous ownership are skipped.
  5. After each confirmed release, the same mission is reopened and rescanned. The individual vehicle-window route is used only when the LSSM controls do not appear.
  6. Select Stop at any time. The Toolkit closes and removes only the MissionChief lightbox layers owned by the sweep before continuing or finishing.
Transport Sweep cannot grant alliance permissions and should be used only when your role and alliance rules allow it. It is manual, sequential, bounded and cancellation-aware.

Settings Backup and Recovery

Move a complete Toolkit setup between browsers or devices.

  • Export All creates a timestamped JSON containing preferences, layouts, profiles, bookmarks and the saved webhook.
  • Import All accepts current and supported legacy Toolkit backups.
  • Reset returns persistent configuration to defaults; it does not uninstall the userscript.
Because exports may contain the Discord webhook, treat them as private credentials.

Performance and Cleanup

How background work is limited and safely removed.

Adaptive refresh

Mission, progress and vehicle sources use separate intervals, minimum request gaps and error backoff.

Caches and signatures

Marker registries, building IDs, heatmap sources, mission snapshots and panel renders avoid unnecessary repetition.

Runtime cleanup

Timers, animation frames, observers, listeners, map bindings, requests and fetch controllers are centrally tracked.

Single-window sweep lifecycle

Patient Transport Sweep claims the exact MissionChief lightbox, iframe and backdrop layers created by each open action, closes them before navigation and removes only sweep-owned remnants.

Alliance Buildings Map Blocker

Suppresses the heavy map on the Alliance Buildings/Courses page and expands the list to full width.

Changing Map Blocker requires a reload. Repair logic restores and resizes the normal map after leaving the blocked page.

Troubleshooting

Practical checks before assuming a feature is broken.

Guide will not load

Use Reload in the Help toolbar. Confirm raw.githubusercontent.com is not blocked. The Toolkit shows an offline fallback rather than a blank view.

Mission data says syncing

Wait for the first personal vehicle or progress request. Syncing is neutral and does not mean zero units.

Map control is missing

Reload MissionChief and confirm only the current Toolkit userscript is enabled for the domain.

Panel is off-screen

Use panel reset on desktop. On touch layouts, switch the mode Off and back to Auto after a major viewport change.

Payout sound is silent

Enable sound and volume, then interact with the page once so the browser can unlock audio.

Alliance page is slow

Enable Map Blocker in Settings and reload. ON means the heavy alliance map is blocked.

Useful diagnostic details

  1. Installed Toolkit version.
  2. Desktop, Tablet or iOS Mobile layout.
  3. Active interface theme and map skin.
  4. Relevant filter and overlay settings.
  5. A screenshot showing the complete affected panel.

Frequently Asked Questions

Common operational and privacy questions.

Does Help send MissionChief data to GitHub?

No. It downloads a static HTML document and does not need account data.

Why is the guide public?

It contains user documentation only. Public hosting lets all users load the same guide without exposing the private migration repository.

Can the guide update without reinstalling?

Yes. Documentation can be updated independently. Changes to the Help viewer or Toolkit controls still require a userscript release.

Does the Toolkit stop working if GitHub is unavailable?

No. Only the external guide may be unavailable; the main Toolkit continues working.

Do themes change operational meaning?

No. Protected semantic colours retain their meaning across all seven interface systems.

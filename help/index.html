01

Quick Start

The shortest route from installation to a working command setup.

  1. Install Tampermonkey or a compatible userscript manager.
  2. Install the Toolkit through its verified Greasy Fork page.
  3. Reload MissionChief and open the Toolkit command button.
  4. Select Desktop, Tablet or iOS Mobile Mode for the current device.
  5. Enable only the operational families needed for the current workflow.
Existing settings are retained across the v5 upgrade line. A manual reset is not required.
02

Operational Window Suite

Four coordinated families, one settings model and one lifecycle per active mission document.

Enhanced Operational Requirements

Reconciles vehicles, equipment, personnel, capacity, trailers, selected units, en-route units, on-scene units and still-needed demand.

Extended Call Window

Adds patient and vehicle summaries, selected and ARR counters, search, highlighting, generation/alarm context, keywords and map controls.

Extended Mission List

Adds sorting, starring, collapsing, patient/prisoner indicators, credit and remaining-time badges, and native sharing controls.

Enhanced Transport Requests

Provides opt-in guarded transport assistance with exact-route validation, candidate filtering, ambiguity rejection and duplicate protection.

Typed settings

Operational settings use switches, number inputs, colours, selects and structured editors. Internal overlay, drag, starred and collapsed state is not exposed as raw JSON.

Source authority

The Toolkit evaluates active native requirement roots, grouped requirement data and compatible hidden LSSM raw-data carriers. Sources are scored by parsed evidence, active mission ownership, visibility and connection state, then rebound when MissionChief replaces the authoritative markup.

A visible equivalent LSSM requirements panel suppresses the matching Toolkit surface. Hidden raw evidence may still be used for reconciliation.

When the live placeholder is empty, v5.0.7 may perform one bounded same-origin mission-page recovery with mission-scoped caching, backoff and rebinding.

04

Extended Call Window

Mission-window intelligence without a competing lifecycle.

  • Patient and vehicle summaries.
  • Selected-unit, player and ARR counters.
  • Permanent vehicle and ARR search.
  • ARR highlighting and configurable colours.
  • Generation time, alarm information and mission keywords.
  • Collapsible patient and vehicle areas with sticky headers.
  • Responsive Desktop, Tablet/iPad and iOS controls.
05

Extended Mission List

Prioritise the live mission list while retaining deterministic state.

  • Sort missions by the configured operational order.
  • Star high-priority missions and collapse lower-priority entries.
  • Show patient, prisoner, average-credit and remaining-time badges.
  • Use MissionChief native share controls.
  • Retain ordering and state through live mission-list refreshes.
06

Enhanced Transport Requests

Transport assistance remains explicitly opt-in and guarded.

  • The current route must match a reviewed transport route.
  • Hidden or disabled candidates are ignored.
  • Ambiguous candidate sets are rejected rather than guessed.
  • One action token is retained per route to prevent duplicate actions.
  • Automatic transport opening remains disabled by default.

Mission command

Triage, inspect and navigate incidents without losing map context.

Mission Age Watch

Filter and sort personal and alliance missions by age, ownership, category, state and distance. Shortcut: 6.

Critical View and Mission Inspector

Concentrate urgent work and load deeper context only when requested.

Mission Value

Shows a verified mission value and hides unavailable values rather than guessing.

Monitoring

Mission Spawn, Stuck Detector, Major Incident Feed, Transport Watcher and Patient Transport Sweep use bounded lifecycle ownership.

08

Fleet, map and finance

Read capability, geography and financial performance from one command layer.

Fleet identity

Vehicle Code Status summarises response codes. Custom Vehicle Badges retain native vehicle types while exposing Own Vehicle Categories.

Map operations

Coverage Heat Map, coverage rings, Smart Bookmark Labels, visibility controls, Focus Mode and Clean Mode improve geographic command.

Resource Gap

Compares mission demand with available personal vehicle context inside a selected radius without assuming alliance availability.

Financial intelligence

Reconciles income, spending and variance; produces graphics and configured Discord summaries from the same canonical model.

11

Configuration and privacy

Most state remains local to the browser.

  • Export settings before substantial browser or device changes.
  • Settings exports can contain a saved Discord webhook and must remain private.
  • Hosted media is loaded only when required by the active interface.
  • Reset only the affected feature where possible; test with Map Command before a full reset.
12

Troubleshooting

Preserve evidence before resetting configuration.

Requirements remain amber

  1. Allow the mission window to finish loading.
  2. Check whether MissionChief exposes requirement evidence.
  3. Check for a visible equivalent LSSM panel.
  4. Record the exact mission and unresolved wording.

A button or panel is missing

  1. Reload MissionChief once.
  2. Confirm the correct operating mode.
  3. Check whether the command bar is collapsed.
  4. Report browser, userscript-manager and Toolkit versions.

Performance changed

  1. Disable unneeded coverage or monitoring overlays.
  2. Allow idle bootstrap to complete.
  3. Compare with the Toolkit disabled.
  4. Include window.__MCMS_STARTUP_METRICS__ in a structured report.
